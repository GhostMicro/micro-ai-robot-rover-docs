    ตารางการวายสายระบบหุ่นยนต์ (Wiring Guide v2.0)

    ตารางสรุปการเชื่อมต่อสายไฟแยกตามส่วนประกอบ 4 ชุดใหญ่ เพื่อความเป็นระเบียบในการไล่วงจร

    🕹️ ชุดที่ 1: Joystick Unit (Remote)

    อุปกรณ์: Arduino Nano + Joystick Module ทำหน้าที่: อ่านค่าจากคันโยกและส่งข้อมูลผ่าน Serial ไปยังส่วน Transmitter

    บอร์ด Arduino Nano อุปกรณ์ Joystick / ปุ่ม หมายเหตุ
    5V VCC
    GND GND ต่อร่วมทุกปุ่ม
    A0 VRx ตรวจสอบแนวแกน X
    A1 VRy ตรวจสอบแนวแกน Y
    D2 Joystick SW ปุ่มกดในตัวคันโยก
    D3 Extra Button 1 ใช้ Internal Pull-up
    D4 Extra Button 2 ใช้ Internal Pull-up
    D5 Extra Button 3 ใช้ Internal Pull-up
    D6 Extra Button 4 ใช้ Internal Pull-up
    TX (ไปที่ Transmitter) ส่งข้อมูลไปยัง ESP32

    �️ วิธีการต่อปุ่มแบบ "ปลอดภัยที่สุด" (High Reliability/Pro Mode)

    หากต้องการความเสถียรสูงสุด ป้องกันสัญญาณรบกวน (Noise) จากมอเตอร์ และป้องกันบอร์ดพังจากการตั้งค่าขา Pin ผิดพลาด แนะนำให้ใช้ R (Resistor) ภายนอก ดังนี้ครับ:

    อุปกรณ์ที่ต้องใช้: * 10k Ohm Resistor (น้ำตาล-ดำ-ส้ม): จำนวน 4 ตัว (ใช้แบบ 2W หรือ 4W ที่คุณมีได้เลยครับ แข็งแรงทนทานดีมาก) * 1k Ohm Resistor (น้ำตาล-ดำ-แดง): จำนวน 4 ตัว (ใช้แบบ 2W หรือ 4W ได้เช่นกัน) * หมายเหตุ: เนื่องจากตัวต้านทาน 2W/4W มีขนาดใหญ่ ตรวจสอบพื้นที่การจัดวางไม่ให้ขาไปแตะกับส่วนอื่นครับ

    ผังการต่อ (ต่อแบบนี้ทั้ง 4 ปุ่ม): 1. ขา 5V Arduino -> ต่อเข้าขาหนึ่งของ R 10k 2. ขาอีกข้างของ R 10k -> ต่อเข้ากับ ขาปุ่มด้านที่ 1 3. ขาปุ่มด้านที่ 1 (จุดเดิมกับข้อ 2) -> ต่อผ่าน R 1k -> แล้วจึงวิ่งเข้า Digital Pin (D3-D6) 4. ขาปุ่มด้านที่ 2 -> ต่อลง GND

    💡 ทำไมวิธีนี้ถึง "ปลอดภัย" กว่า?

    1. ป้องกันบอร์ดพัง (Current Limiting): หากคุณเผลอเขียนโปรแกรมตั้งค่าขา Pin เป็น OUTPUT และสั่งให้เป็น HIGH ในขณะที่กดปุ่มอยู่ R 1k จะช่วยจำกัดกระแสไม่ให้ไฟช็อตลงกราวด์โดยตรงจนชิปไหม้
    2. สู้สัญญาณรบกวนได้ดีกว่า (Noise Immunity): R 10k แบบภายนอกมีความต้านทานที่ "แน่น" (Stronger Pull-up) กว่าแบบในตัวชิป ทำให้โอกาสที่หุ่นยนต์จะกดปุ่มเองเพราะสัญญาณรบกวนจากมอเตอร์มีน้อยลงมาก

    3. ความเสถียร: สัญญาณจะนิ่งกว่ามากในสภาพแวดล้อมที่มีการส่งสัญญาณวิทยุ (Transmitter) อยู่ใกล้ๆ

    4. ไม่กด: อ่านได้ HIGH (1)

    5. กด: อ่านได้ LOW (0)

    🚀 การอัปเกรด Joystick (Analog Smoothing)

    • C Ceramic (104/0.1uF): ต่อคร่อมขา A0 กับ GND และ A1 กับ GND เพื่อกรองสัญญาณรบกวนความถี่สูง (Noise) จากตัวต้านทานปรับค่าได้ของ Joystick ทำให้ค่าที่อ่านได้นิ่งขึ้นมาก
    • Decoupling: ต่อ C Electrolytic (100uF) และ C Ceramic (104) คร่อม VCC/GND ของ Nano 1 ชุด เพื่อกันไฟกระชากตอนกดปุ่ม

    📡 ชุดที่ 2: Transmitter Unit (Station)

    อุปกรณ์: ESP32 + Level Converter + OLED 0.96 ทำหน้าที่: รับค่าจาก Joystick Nano มาแสดงผลและส่งสัญญาณไร้สาย (WiFi/ESP-NOW)

    อุปกรณ์ต้นทาง อุปกรณ์ปลายทาง หมายเหตุ
    ESP32 3.3V OLED VCC / LLC LV จ่ายไฟ 3.3V
    ESP32 GND OLED GND / LLC GND กราวด์ร่วม
    ESP32 GPIO 21 OLED SDA I2C Data
    ESP32 GPIO 22 OLED SCL I2C Clock
    LLC HV1 Joystick Nano TX รับสัญญาณ 5V
    LLC LV1 ESP32 GPIO 16 (RX2) แปลงเป็น 3.3V เข้า ESP32
    External 5V LLC HV จ่ายไฟ 5V ให้ฝั่ง High

    🚀 การอัปเกรด Transmitter (Radio Stability)

    • C Electrolytic (220uF หรือมากกว่า): ต่อคร่อมขา 3.3V และ GND ของ ESP32 (สำคัญมาก) เพราะเวลา ESP32 ส่งข้อมูล WiFi/ESP-NOW จะกินกระแสแบบกระชากชั่วขณะ หาก C ไม่พอจะทำให้บอร์ด Reboot เอง
    • C Ceramic (104): ต่อคร่อมขา 3.3V/GND ให้ใกล้ชิดกับตัวเครื่องมากที่สุดเพื่อกรองสัญญาณรบกวน

    🧠 ชุดที่ 3: Receiver Head (Robot Head)

    อุปกรณ์: ESP32 + STM32 + Arduino Nano (Sensor) + ESP32-CAM + Level Converter ทำหน้าที่: "ศูนย์รวมความรู้สึก" ทั้งการมองเห็น (Vision), การรับสัญญาณจากรีโมท, ประมวลผลโดย STM32 และดึงค่าเซนเซอร์

    3.1 การเชื่อมต่อ Serial / Vision / Data Log

    อุปกรณ์ต้นทาง อุปกรณ์ปลายทาง หมายเหตุ
    ESP32 TX2 (17) STM32 RX2 (PA3) รับคำสั่ง Remote (3.3V)
    ESP32 RX2 (16) STM32 TX2 (PA2) ส่ง Telemetry (3.3V)
    External 5V ESP32-CAM 5V จ่ายไฟ 5V แยก (กันกวน)
    Common GND ESP32-CAM GND กราวด์ร่วม

    3.2 การต่อ MicroSD Card Module (Black Box)

    ใช้สำหรับบันทึก Log การทำงาน, ความเร็ว, และสถานะเซนเซอร์แบบ Offline

    ESP32 Pin (Main) MicroSD Module หมายเหตุ
    5V / 3.3V VCC เช็คแรงดันโมดูลคุณ (ส่วนใหญ่ 5V)
    GND GND
    GPIO 18 SCK SPI Clock
    GPIO 19 MISO SPI Data In
    GPIO 23 MOSI SPI Data Out
    GPIO 5 CS Chip Select

    3.2 การใช้ ESP32-CAM ในส่วนหัว (Perception)

    • Vision: ส่งภาพวิดีโอสดไปยัง Dashboard ผ่าน WiFi
    • Audio (ถ้ามี): ใช้ไมโครโฟนบนบอร์ดหรือต่อเพิ่มเพื่อฟังเสียงรอบข้าง
    • Flash Light: ใช้เป็นไฟนำทางในที่มืดได้ (คุมผ่านโปรแกรม)

    3.2 การใช้ Level Converter (LLC) ระหว่าง STM32 (3.3V) และ Nano (5V)

    เนื่องจาก STM32 ทำงานที่ 3.3V และ Nano ทำงานที่ 5V จึงต้องใช้ LLC คั่นกลาง

    ฝั่ง STM32 (Low Voltage 3.3V) LLC Pin ฝั่ง Nano (High Voltage 5V) หมายเหตุ
    STM32 3.3V LV Nano 5V (HV) แหล่งจ่ายไฟอ้างอิง
    STM32 GND GND Nano GND กราวด์ร่วม
    STM32 TX1 (PA9) LV1 <--> HV1 Nano RX (D0) ส่งคำสั่งไป Nano
    STM32 RX1 (PA10) LV2 <--> HV2 Nano TX (D1) รับค่าจาก Nano

    3.3 การต่อเซนเซอร์ที่บอร์ด Nano (Sensor Node)

    Arduino Nano เซนเซอร์ หมายเหตุ
    D12 Ultrasonic TRIG
    D11 Ultrasonic ECHO
    A4/A5 MPU6050 (SDA/SCL) ถ้ามีการใช้ Gyro

    🚀 การอัปเกรด Receiver Head (Logic Protection & Power)

    • R 4.7k (Pull-up): หากใช้สาย I2C ยาว ให้ต่อ R 4.7k ดึงขา SDA/SCL ขึ้น 3.3V (STM32) เพื่อให้การสื่อสารกับ Gyro และ Sensor นิ่งขึ้น
    • MicroSD Black Box: ตั้งโปรแกรมให้บันทึกค่า L_Speed, R_Speed, Volt, Dist ลงในไฟล์ .txt ทุกๆ 1 วินาที เพื่อเก็บไว้ดูย้อนหลังหากหุ่นยนต์เกิดปัญหา (Crash Analysis)
    • C Electrolytic (470uF/10V ขึ้นไป): ต่อคร่อมขา 5V และ GND ของ ESP32-CAM (สำคัญที่สุด) เพื่อป้องกันอาการภาพกระตุกหรือบอร์ดรีเซ็ตเวลาส่งภาพ
    • Decoupling Stack: เนื่องจากมีทั้ง ESP32 (x2) และ STM32 ให้ต่อ C 100uF + C 104 แยกกันที่ขาไฟเลี้ยงของบอร์ดแต่ละตัว เพื่อไม่ให้สัญญาณกวนกันเอง

    🏎️ ชุดที่ 4: Body Driver (Robot Base)

    อุปกรณ์: Arduino Nano + Motor Driver (L298N) + Motors ทำหน้าที่: รับผลลัพธ์การคำนวณจาก Brain มาควบคุมการหมุนของล้อ

    บอร์ด Arduino Nano อุปกรณ์ L298N / Motor หมายเหตุ
    D10 (PWM) ENA ควบคุมความเร็วซ้าย
    D9 IN1 ทิศทางซ้าย 1
    D8 IN2 ทิศทางซ้าย 2
    D7 IN3 ทิศทางขวา 1
    D6 IN4 ทิศทางขวา 2
    D5 (PWM) ENB ควบคุมความเร็วขวา
    GND GND (ร่วมกับ STM32/Battery) สำคัญมาก!
    RX (จาก STM32 PB10/TX3) รับคำสั่งความเร็ว

    🚀 การอัปเกรด Body Driver (High Power & Safety)

    นี่คือจุดที่อุปกรณ์ที่คุณมี (C, R, TR) จะโชว์ประสิทธิภาพได้ดีที่สุดครับ: * C 1000uF/25V (Electrolytic): ต่อคร่อมขั้วไฟเข้า 12V/7.4V ของ L298N เพื่อรับแรงกระแทกตอนมอเตอร์เริ่มหมุน * C 104 (Ceramic) x3 ต่อมอเตอร์: ต่อขาหนึ่งเข้าขั้วมอเตอร์ + อีกข้างลงโครงมอเตอร์ (ทำทั้ง 2 ขั้ว) และต่อคร่อมขั้วบวก/ลบ อีก 1 ตัว เพื่อฆ่าสัญญาณรบกวน (Spark) จากแปรงถ่านมอเตอร์ (แนะนำอย่างยิ่ง) * Transistor (TR) (เช่น NPN 2N2222/TIP120): * ใช้เป็น Switch สำหรับไฟหน้า (Headlight) หรือ พัดลมระบายอากาศ L298N * วิธีต่อ: ขา Base ต่อผ่าน R 1k เข้า Arduino, ขา Collector ต่อผ่านโหลด (ไฟ/พัดลม) เข้า 5V, ขา Emitter ลง GND * Diodes (Flyback): หาก L298N ของคุณไม่มีไดโอด ให้ต่อคร่อมขั้วมอเตอร์ (ต่อสวนทางกับไฟ) เพื่อกันไฟย้อนกลับมาเผาบอร์ดตอนเบรกมอเตอร์

    [!WARNING] Common Ground: อย่าลืมเชื่อมต่อขา GND ของทุกชุดอุปกรณ์เข้าด้วยกัน เพื่อให้สัญญาณสื่อสาร (Serial/I2C) ทำงานได้อย่างถูกต้อง

    🛠️ การจัดการ Firmware และการอัปเดต (Maintenance Guide)

    เพื่อให้คุณไม่ต้องถอดบอร์ดเข้า-ออกเวลาแก้ไขโปรแกรม แนะนำ 2 วิธีการแยกสัญญาณ (Isolation) ดังนี้ครับ:

    1. วิธีใช้ "ตัวต้านทาน 1k" (แนะนำ - สะดวกที่สุด)

    • การต่อ: ใส่ R 1k อนุกรมคั่นที่สายสัญญาณระหว่างบอร์ดกับ LLC ดังนี้ครับ:
      • ฝั่ง Nano: ต่อคั่นระหว่าง [Nano D0, D1] <---> [R 1k] <---> [LLC HV1, HV2]
      • ฝั่ง STM32: (ถ้าใช้ Serial อัปเดต) ต่อคั่นระหว่าง [STM Pin] <---> [R 1k] <---> [LLC LV1, LV2]
    • หลักการ: เมื่อคุณเสียบสาย USB สัญญาณจากชิป USB บนบอร์ดจะ "แรงกว่า" เพราะมันไม่ต้องวิ่งผ่านตัวต้านทาน ทำให้มันสามารถส่งข้อมูลลงชิปได้โดยไม่ถูกสัญญาณจาก LLC มารบกวนครับ

    2. วิธีใช้ "สวิตช์เลื่อน" (Slide Switch - มั่นใจที่สุด)

    • การต่อ: ใช้สวิตช์เลื่อนแบบ 2 ทาง (DPDT) คั่นสาย TX และ RX เพื่อเลือกสลับระหว่าง "โหมดทำงาน" กับ "โหมดโปรแกรม"
    • ข้อดี: ตัดสัญญาณขาดจากกัน 100% ปลอดภัยแน่นอน
    • ข้อเสีย: ต้องวายสายเพิ่มเยอะขึ้น และต้องอย่าลืมเลื่อนสวิตช์ก่อนอัปเดต ถ้าลืมเลื่อนแล้วกดอัปโหลด โปรแกรมจะฟ้อง Error ครับ
    • ผลต่อสัญญาณ: สวิตช์เชิงกลไม่มีผลต่อคุณภาพสัญญาณดิจิทัลในความเร็วนี้ครับ แต่ถ้าจุดบัดกรีไม่แน่นอาจจะหลวมได้ง่ายกว่าแบบแรก

    3. ระบบ OTA สำหรับ ESP32

    • สำหรับ ESP32 ไม่จำเป็นต้องใช้สวิตช์หรือ R คั่น เพราะเราจะอัปเดตผ่าน WiFi (OTA) โดยตรง ซึ่งจะข้ามผ่านระบบ Serial ไปเลยครับ

    ✅ Checklist ก่อนลงบอร์ด (Final Pre-Assembly)

    เพื่อให้การจัดวางอุปกรณ์ (Layout) บนบอร์ดทำได้ทีเดียวจบ แนะนำให้เช็คอุปกรณ์ที่จะ "ฝัง" ลงในแต่ละบอร์ดดังนี้ครับ:

    บอร์ด 🕹️ Joystick (ที่รีโมท)

    • [ ] Resistors: 10k x4 (Pull-up), 1k x4 (Protect), 1k x2 (Serial Isolation)
    • [ ] Capacitors: Ceramic 104 x2 (คร่อมขา Analog X,Y), Ceramic 104 x1 + Electrolytic 100uF x1 (กรองไฟเลี้ยง)

    บอร์ด 📡 Transmitter (ที่สถานี/รีโมท)

    • [ ] Capacitors: Electrolytic 220uF-470uF x1 (วางใกล้ขาไฟ ESP32 ที่สุด), Ceramic 104 x1
    • [ ] OTA: เตรียมความพร้อมสำหรับ Flash Program ผ่าน WiFi

    บอร์ด 🧠 Receiver Head (บนหัวหุ่น)

    • [ ] Components: ESP32, STM32, Nano, ESP32-CAM, LLC, MicroSD Module
    • [ ] Decoupling: Ceramic 104 x5 + Electrolytic 100uF x3 + Electrolytic 470uF x1 (สำหรับ CAM)
    • [ ] Wiring: เตรียมสายไฟ 6 เส้นสำหรับ SPI ของ SD Card และพิน SWD สำหรับ STM32

    บอร์ด 🏎️ Body Driver (ฐานหุ่น)

    • [ ] Power: Capacitor 1000uF ขึ้นไป วางใกล้ขั้วไฟเข้า L298N
    • [ ] Motor Noise: Capacitor 104 x6 (บัดกรีติดที่ขั้วมอเตอร์โดยตรงทั้ง 3 จุดต่อตัว)
    • [ ] Extra: Transistor + R 1k สำหรับสวิตช์ไฟหน้าหรือพัดลม
    • [ ] Fuse: ติดตั้งจุดที่เปลี่ยนง่ายที่สุด (3A-5A)

    🛡️ สรุปอุปกรณ์เสริมเพื่อความปลอดภัยและประสิทธิภาพ (Pro Upgrades)

    1. ระบบจัดการพลังงานและความแม่นยำ (Advanced Power & Protection)

    • Dual Rail Strategy: แยก Buck Converter เป็น 2 ตัว (Logic และ Motor) เพื่อกันไฟกวน
    • Star Grounding: รวมสาย GND ทุกเส้นมาที่จุดเดียวที่ขั้วลบแบตเตอรี่
    • Zener Diode (5.1V): ต่อคร่อมขา 5V/GND ของชุดสมองเพื่อป้องกันแรงดันเกิน (Voltage Spike)
    • Reverse Polarity Diode: ต่ออนุกรมที่ขั้วบวกก่อนเข้า Buck เพื่อกันการเสียบแบตฯ สลับขั้ว

    2. วงจรเสริมเพื่อการตรวจสอบ (Diagnostic & Monitoring)

    • Battery Monitor: ขั้วบวกแบต -> [R 10k] -> [จุด A] -> [R 4.7k] -> GND (ลากจุด A ไปเข้า ESP32 GPIO 34)
    • Heartbeat LED: ต่อ LED ผ่าน R 220 Ohm ที่ขา Digital ที่ว่างของทุกบอร์ด เพื่อดูสถานะการทำงาน
    • Inductor (LC Filter): ใช้ขดลวดต่ออนุกรมกับขาไฟเข้า ESP32-CAM เพื่อกรองสัญญาณรบกวนความถี่สูง

    3. การจัดการสัญญาณรบกวนคลื่นวิทยุ (RF Optimization)

    • Physical Separation: วาง ESP32 Main และ ESP32-CAM ห่างกันอย่างน้อย 5-10 ซม.
    • Antenna Orientation: วางบอร์ดหนึ่งแนวตั้ง อีกบอร์ดแนวนอน (Cross-polarization)
    • Channel Lock: ตั้งค่า WiFi กล้อง (CH 1) และ ESP-NOW (CH 11) ให้ห่างกันที่สุด

    4. ข้อแนะนำการติดตั้งตัวต้านทานใหญ่ (2W/4W)

    • บัดกรีให้แน่น: เนื่องจากตัว R มีน้ำหนักมากกว่าปกติ ให้ใช้กาวร้อน (Hot Glue) ช่วยยึดตัวถัง R
    • แยกขาให้ห่าง: ระวังขาของ R 2W/4W ที่หนาและแข็งไปชนกับขาอุปกรณ์ข้างเคียง
